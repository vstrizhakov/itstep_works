		UML - Унифицированный язый програм. (универсальн язык между разработчиком и заказчиком - в графическом виде), нужен для моделирования 
	Состоит из 4х эл нотаций (диаграммы состоят):
1) Фигуры
2) линии
3) Значки
4) Надписи 

Главная ЗАДАЧА UML - предоставить модель (набор диаграмм)
	
		ВИДЫ ДИАГРАММ

1.	USE Case - Диаграмма ПРЕЦЕДЕНТОВ или СЛУЧАЕВ ИСПОЛЬЗОВАНИЯ	- это то что программа должна делать (начало для создания других диаграмм - т е первая модель)			

2.	Class - Диаграмма КЛАССОВ

3.	Obgect - диаграмма объектов

4.	Sequence - диаграмма последовательностей

5.	Collabration - диаграмма взаимодействия (кооперации)

6.	Statechart - диаг состояния

7. 	Activity - диаг деятельности (актвности)

8.	Deployment - диагр развёртывания


		USE Case 
	В элепсе пишуться USE Case (например для онлайн библиотеке: 1)Регистрация; 2) Просмотр содерж; 3) Заказ; 4) предзаказ)
Прецидент - описание отдельного оспекта поведения пользователя
USE Case  - не оисывает как достигается результат он его фиксирует
